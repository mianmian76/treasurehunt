# 寻箱决策平台 — 设计理念

## 选定方案：「古典探险地图」风格

**Design Movement**: 复古探险 × 现代数据可视化的融合美学（Retro-Expedition meets Modern Data）

**Core Principles**:
1. 地图即界面——整个平台以网格地图为核心视觉元素，操作围绕地图展开
2. 信息层次清晰——感知数据、候选点、路径建议三层信息用颜色和动画区分
3. 决策透明化——每一步推理过程可视化，让玩家理解AI的思考逻辑
4. 探险感与精确感并存——温暖的羊皮纸色调 + 精确的网格线条

**Color Philosophy**:
- 背景：深海军蓝 `oklch(0.18 0.04 240)` — 沉浸感，像夜间航海图
- 地图格子：深蓝灰 `oklch(0.25 0.03 240)` — 可通行节点
- 主色调：琥珀金 `oklch(0.75 0.15 65)` — 宝藏、高亮、CTA
- 感知信号色：翡翠绿 `oklch(0.65 0.18 160)` — 距离1/2/3的热力渐变
- 路径色：天蓝 `oklch(0.65 0.15 220)` — 建议路径
- 危险/排除色：珊瑚红 `oklch(0.65 0.18 25)` — 已排除候选点

**Layout Paradigm**:
- 左侧：交互式地图（占60%宽度），网格可点击
- 右侧：决策面板（占40%宽度），分为感知输入、推理分析、建议移动三个折叠区
- 顶部：进度条 + 步数计数器 + 宝藏状态

**Signature Elements**:
1. 地图节点的脉冲动画（感知到信号时节点发光）
2. 路径动画（建议路径以虚线动态绘制）
3. 贝叶斯过滤过程的实时可视化（候选点逐渐变暗/消失）

**Typography System**:
- 标题：`Cinzel` — 古典衬线，探险感
- 界面文字：`IBM Plex Mono` — 等宽字体，数据/坐标感
- 正文：`Noto Sans SC` — 中文可读性

**Animation**:
- 节点发现：金色粒子爆炸效果
- 路径绘制：虚线从起点到终点逐步延伸
- 候选点排除：节点透明度渐变为0.2
- 感知信号：同心圆扩散波纹
